Java学习笔记：面向对象基础

发表于 2023-04-23 更新于 2023-04-24 分类于 笔记 ， Java

类

带有main方法，可执行的类叫做测试类。
不带main方法，由其他类调用执行的类就是JavaBean类。
标准JavaBean类编写要求如下。

类名须正确描述类型，驼峰命名
成员变量使用private修饰符
定义无参和带全部参数的两种构造方法
定义成员方法

构造方法（构造器、构造函数）

  • 构造方法的方法名与类名一致
  • 没有返回值类型，不加void
  • 没有具体返回值，不能返回（return）
  • 参数可为空

构造方法不能手动调用，是在创建对象时由虚拟机自动调用。
每创建一次对象就会调用一次方法。
如果没有定义构造方法，虚拟机会自动加上一个无参构造方法。

如果没有定义无参构造方法，而定义了有参构造方法，虚拟机将不提供默认构造方法。
有参构造方法和无参构造方法的方法名相同，但参数不同，这是构造函数的重载。

继承

继承的概念允许创建分等级层次的类，可以让一个类（子）继承另一个类（父）的属性和方法，并允许在此基础上修改。

Java只支持单继承，不支持多继承，但是支持多层继承。
即一个子类不能继承多个父类，但是子类继承的父类也可以继承其他父类，父类继承的父类称为子类的间接父类。
每一个类都直接或间接继承于Object。

当父类中的方法不能满足子类中的需求，可以对方法进行重写。
在子类中出现与父类中一模一样的方法声明，称之为重写方法，发生重写就会覆盖父类的方法。

重写方法时，方法的名称和形参列表要和父类一致，子类重写父类方法时，子类方法访问权限须大于等于父类（空着不写<protected<public）。
子类重写父类方法时，返回值类型须小于等于父类。
只有被添加到虚方法表中的方法才能被重写。
方法重写的本质就是子类覆盖了虚方法表中的方法。

继承规则

子类只能访问父类中非私有的成员。
父类的构造方法无论是否私有，子类都不能继承。
成员变量无论是否私有，子类都可以继承，但有private成员变量可能无法使用。
成员方法非私有能继承，私有不能继承。
子类继承父类的虚方法表，虚方法即:非private修饰，非static修饰，非final修饰。
即使不继承static方法也可直接调用。

静态方法

静态方法通常用在测试类和工具类中。
工具类将会使用private修饰构造方法，因为工具类不是像JavaBean类一样描述某类事物的类，而是提供方法使用的类，私有化构造方法可以让外界无法创建这个类的对象。

静态方法只能访问静态变量和静态方法。
而非静态方法可以访问静态和非静态。
静态方法中没有this关键字。

静态属性的定义

多态

现有父类Person，属性有name和age，方法有showInfo。两个继承于它的子类Teacher和Student，且重写了showInfo方法。

现定义以下方法

在register方法中的参数为Person类型，由于Teacher和Student都是Person的子类，所以register方法中的参数也可以为Teacher和Student。
这就是多态的体现，对象的多种形态。
多态的前提是：有继承/实现关系，有父类引用指向子类对象，有方法重写。
使用父类作为参数，即可接收所有子类对象，这体现了多态的便利与扩展性。

可以使用多态的方式来创建对象

调用成员变量时编译看左边(父类)，运行也看左边(父类)。
在编译时会看左边父类代码中是否有该变量，有则编译成功，没有就编译失败。
运行时实际获取的将会是父类中的成员变量。
如
p1.className// Person

调用成员方法时编译看左边(父类),但是运行看右边(子类)。
在编译时，如果左边父类没有这个方法，则编译失败，有则编译成功。
在运行时，实际上运行的会是子类中的方法。
p1.showInfo(); //Student.showInfo()

使用p1调用成员变量和成员方法，p1是Person类型的，所以默认会从Person中查找
成员变量:子类对象会把父类的成员变量继承下来，所以有Person: className和Student:className。
p1是Person类型，所以会调用Person:className;
成员方法:子类继承父类后如果对方法进行重写(Override)，重写的子类方法将会覆盖虚方法表中的父类方法，所以调用方法时是子类中的方法。

p1.showStuInfo();
无法运行
这是多态的弊端，不能调用子类中特有的方法。
解决方法是将父类型变为子类型,这样就可以使用子类特有方法。

转换类型时不能随意转换为其他类型
如
Teacher t1=(Student) s1;
两者不能互相转换，可以使用条件判断是否可以转换。

抽象类

抽象类不能实例化，也就是不能使用抽象类创建一个对象。
但抽象类也可以有构造方法，抽象类本身不能创建对象，但是继承抽象类的子类可以创建，子类创建时可以使用抽象类的构造方法。
有抽象方法的类一定是抽象类，但是抽象类中不一定要有抽象方法。
抽象类的子类必须重写抽象类中的所有抽象方法，除非子类也是抽象类。

接口

接口是对行为的抽象。
接口定义格式如下，接口不能被实例化，接口和类之间是实现关系。
须用implements关键字表示。
接口的子类(即实现类)，必须重写接口中的所有抽象方法，除非子类是抽象类。
接口和类的实现关系可以是单实现，也可以是多实现，即一个类可以实现多个接口，同时还能再继承一个类。
没有构造方法。
接口中的成员变量只能是常量，默认修饰符为public static final

  • 被final修饰的方法表明该方法为最终方法，不能被重写。
  • 修饰类表示最终类，不能被继承。
  • 修饰引用类型时，地址不能改变，但地址内部数据可以改变。
  • 修饰变量则变量只能且必须被赋值一次，作为常量。

当一个类继承的两个接口中有着相同名字的方法，只需重写一次就等于将两个同名方法重写。
接口是对行为的抽象，需要让类拥有行为，就让类实现对应的接口。
当方法传入参数为接口时，则参数可以填入所有实现该接口的类的对象，这是接口的多态。

&emsp;&emsp;首先让我们来安装git安装nodejs安装npm以及安装之后用来连接云服务器的finalshell吧。

安装完git后通过右键直接打开git bash，且git自带ssh（Secure Shell），通过输入ssh可以查看。

$ ssh
usage: ssh [-46AaCfGgKkMNnqsTtVvXxYy] [-B bind_interface]
       [-b bind_address] [-c cipher_spec] [-D [bind_address:]port]
       [-E log_file] [-e escape_char] [-F configfile] [-I pkcs11]
       [-i identity_file] [-J [user@]host[:port]] [-L address]
       [-l login_name] [-m mac_spec] [-O ctl_cmd] [-o option] [-p port]
       [-Q query_option] [-R address] [-S ctl_path] [-W host:port]
       [-w local_tun[:remote_tun]] destination [command [argument ...]]

输入这个命令采用RSA算法生成密钥和公钥，生成的文件在users/用户/.ssh 目录下，使用记事本打开文件id_rsa.pub，并复制内容添加到github网站中的Settings->SSH and GPG keys->new SSH key
使用公钥授权的方式，在使用git上传文件时就不需要每次登录输入口令了。

ssh-keygen -t rsa

通过在git bash中输入指令

ssh -T git@github.com

进行检验，如果警告是否确定继续连接，只需输入yes同意。
